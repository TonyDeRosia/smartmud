# Adventurer's Lair Player-Runtime Parity Matrix

This matrix records behavior studied from the classic Adventurer's Lair/TBA-style command flow and the repaired Smart MUD equivalents. It is a behavioral reference only; no C source, structs, global lists, file formats, or command-table implementation was ported.

| Area | Adventurer's Lair behavior | Smart MUD current behavior | Smart MUD repaired behavior | Canonical Smart MUD service | Remaining intentional difference |
|---|---|---|---|---|---|
| Room LOOK | `look`/`l` renders room, visible actors, objects, exits. | Plain `l` rendered room. | Preserved canonical room renderer for `look`/`l`. | `MudRuntime._current_room`, `render_room` | HTML semantic roles are retained internally. |
| Character LOOK | Direct look at character/mobile shows description and condition. | Targeted look could fall through or return empty. | Targeted look resolves visible players/NPCs/mobs and renders description, condition, posture/status, equipment. | `_resolve_interaction_target`, `_render_examination` | Smart MUD uses condition bands, not exact HP. |
| Object LOOK | Distinct room/inventory/examine descriptions. | Some target args were lost or hidden behind features. | Item look searches inventory, equipment, and room instances and supports examine descriptions. | Item instances, `_look_item`, `_render_examination` | Field names are JSON/Builder-native. |
| Direction LOOK | `look north` inspects exit. | Could be treated as an ordinary keyword. | Directions resolve before generic objects and render exit text/fallback. | Canonical exits, `_resolve_interaction_target` | Destination details remain Builder-authored. |
| Container LOOK | `look in <container>` lists accessible contents or natural empty/closed text. | Some non-container fixtures produced unhelpful failures; corpses already had partial support. | `look in/inside` preserves container argument and uses canonical corpse/container contents. | `_look_in_container`, item ownership | Non-container chest placeholders remain compatible with old tests. |
| Inventory | Lists carried item short names. | Worked for normal items but could include bad fixtures after GET ALL. | Non-portable authored fixtures are excluded from acquisition and repaired by portability rules. | Item ownership service | Existing corrupted saves are repaired conservatively when loaded through templates. |
| Equipment | Lists equipped items. | Existing behavior preserved. | Existing behavior preserved. | Equipment ownership slots | None. |
| GET | Takes portable room objects only. | Fixed fixtures could be materialized as portable items. | `get <target>` rejects fixed features and nonportable templates. | `pickup_item`, item templates/features | Message wording is Smart MUD-native. |
| GET ALL | Takes all legal portable room objects. | Could pick up Notice Board. | Filters to portable item instances only. | `bulk_get`, item template portability | Does not spam errors for fixed features. |
| DROP | Drops carried portable items to the room. | Could drop an illegally-acquired feature. | Drop path only resolves carried instances; fixed features cannot enter inventory. | `drop_item`, `transfer_item` | Existing invalid historical items are blocked by template portability. |
| DRINK | Drinks from carried drink containers or authored sources. | Clean Water Flask returned undrinkable. | Carried drink containers resolve through consumable profiles and decrement servings. | SurvivalNeedsService, `_drink_from_target` | If needs are unavailable, item action still succeeds for authored drinkables. |
| READ | Readable boards/signs show text. | Notice Board could be looked at but not read through content. | Notice Board remains fixed and exposes authored readable text. | `_resolve_interaction_target`, item template `readable_text` | Board content is concise starter text. |
| CONSIDER | Resolves living targets only. | Dead/living mismatch could occur. | Dead mobs reconcile out of visible living resolution. | Entity state reconciliation, combat foundation | Combat math remains Smart MUD-native. |
| DIAGNOSE | Resolves target condition. | Same target-state risk. | Shares reconciled visible target state. | Entity payload reconciliation | None. |
| KILL | Attacks living visible target; dead target is not living. | A displayed wolf could be “already dead.” | HP/state mismatch excludes dead mobs from room rendering and living target resolution. | CombatRuntimeService, `is_entity_visible` | Respawn policy remains Smart MUD lifecycle-based. |
| CAST | Spell invocation remains explicit. | Multiword abilities were routed as CAST. | `set camp` and `build campfire` parse as abilities/use, not cast. | Parser, AbilityExecutionService | CAST remains for spells. |
| Skill/ability use | Learned skills can be invoked by their command words. | Listed camp skills returned generic failure. | Multiword skill names resolve by longest match and call campsite/campfire services. | AbilityExecutionService, survival camp services | Skills display includes simple Ready status. |
| Numbered targets | `1.wolf`, `2.corpse` choose ordered matches. | Existing keyword helpers supported partial numbering. | Preserved and used through canonical resolver. | `resolve_item_keywords`, `resolve_entity_keywords` | Ordering is stable SQLite/runtime ordering. |
| Ambiguity responses | Ambiguous targets ask for clarification. | Some paths silently fell through. | Ambiguity returns a natural non-empty response. | `_resolve_message`, target resolver | Exact wording is Smart MUD-native. |
| Dead target behavior | Dead mobiles are corpses, not living targets. | Dead mobs could appear living. | Entity payload reconciliation enforces HP/state/is_alive invariant before render/target. | `_entity_payload`, `is_entity_visible`, corpse service | Does not destructively reset player DBs. |
| Portable/fixed objects | Permanent fixtures cannot be taken. | Notice Board could be acquired. | Fixtures use explicit/non-inferred `portable:false` and are filtered from GET ALL. | Item templates, room features, `pickup_item` | Some legacy fixture item templates remain inspectable for compatibility. |
