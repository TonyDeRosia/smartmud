# Phase 13C3-A3F Adventurer's Lair Combat Parity Audit

Primary reference: TonyDeRosia/tbamud_adventurers_lair `src/fight.c`, `src/act.offensive.c`, `src/spells.c`, `src/handler.c`, `src/class.c`, `src/constants.c`, and related headers. This audit records gameplay meaning rather than copying C data, structures, formulas, or file formats.

| Feature | Adventurer's Lair player-facing behavior | Gameplay meaning | Current Smart MUD equivalent | Missing before this phase | Phase plan / implementation | Intentional differences |
|---|---|---|---|---|---|---|
| Entering combat | `kill`/offensive commands place attacker and victim in a fight; followers can assist. | Combat has a persistent opponent relation and room-visible start messaging. | `CombatRuntimeService` encounters, participants, targets, queue. | Opening attacks could bypass a canonical request object. | Player/NPC starts build `CombatActionRequest` and enter `submit_action`. | SQLite encounter rows replace in-memory linked combat lists. |
| Leaving combat | Flee/death/invalid target stops fighting and clears opponent references. | No actor should keep swinging at an absent/dead target. | Participant status and encounter end rows. | Lifecycle side effects were split across combat helpers. | `RuntimeLifecycleService` coordinates defeat and combat-end once. | Restart cancels active encounters conservatively. |
| Attack commands | Basic attacks show hit/miss/damage prose and condition changes. | Commands request an action; resolver owns rolls and damage. | `CombatEngine.resolve_attack`. | Commands/rounds called resolver directly. | `submit_action` is the normal gateway. | Diagnostics are JSON-backed for builders. |
| Round timing | Global violence pulse advances combat and NPC attacks. | Readiness/initiative decide action order, not damage math. | `process_due_rounds` and action queue. | Queued rows consumed then direct-resolved. | Queue consumption creates canonical action requests. | World-minute pulse is Smart MUD's clock. |
| NPC attacks | Mobs fight back and continue attacking in later pulses. | NPC stats/affects/equipment feed the same attack resolver. | Entity actors are converted to canonical `Actor`. | Separate start path. | NPC path uses request gateway. | Advanced AI remains out of scope. |
| Death/defeat | Victim reaches dead/incapacitated states; combat stops. | One transition gates corpse/rewards/cleanup. | Health persistence and corpse helper existed. | Exactly-once transition state was incomplete. | `RuntimeLifecycleService` persists `actor_lifecycle_transitions`. | Smart MUD treats NPC lethal combat as death; PC death remains conservative defeat/death record. |
| Corpses/loot | Corpse is created with eligible belongings/loot. | Corpse identity is stable for the death. | `MudRuntime.create_corpse`. | Combat created corpses directly. | Stable transition id drives one corpse claim. | Existing item policy is reused; no copied AL object formats. |
| Rewards/XP/kill credit | Killers/groups receive rewards after mob death. | Reward handoff belongs to lifecycle, not damage formulas. | Contribution columns and future reward hooks. | No durable reward claim. | Transition stores `reward_claim_id` exactly once. | No new progression economy invented. |
| Respawn | Dead mobs can be restored by world lifecycle. | Respawn schedule is keyed to the killed instance lifecycle. | Actor/entity lifecycle manager hooks. | No transition-keyed durable claim. | Transition stores `respawn_id` exactly once. | Existing respawn timing remains authored by Smart MUD. |
| Saving throws | Spells/effects may be resisted/partial. | Save outcomes alter effect/damage application. | Combat resolution has save diagnostics. | Ability routing did not consistently expose save results. | Ability audit documents canonical target service; request model carries save definition. | FormulaEngine handles Smart MUD formulas. |
| Damage/healing spells | Spells route to damage/healing and messages. | Ability definitions request canonical mutation. | `AbilityExecutionService`, `RuntimeResourceService`, `CombatEngine`. | Direct ability damage component used internal attack call. | Damage can be represented as `CombatActionRequest`; healing remains resource-service canonical with no dead resurrection. | No broad spell library added. |
| Affects/DOT/resources | Affects change stats or tick damage; costs consume resources. | Effects are runtime instances; resources are versioned. | Runtime effects/resources exist. | DOT/environmental sources needed canonical submission. | Request schema includes source/effect IDs; audit documents routing. | Direct dictionary mutation is phased out where service exists. |
| Messages/diagnostics | Damage messages and builder debug text are visible to authorized users. | Players get prose; builders inspect mechanics. | Outbound combat messages and `combatbreakdown`. | History/action/target/formula detail incomplete. | Bounded history table is used by completed subcommands. | Ordinary players cannot see hidden diagnostics. |
