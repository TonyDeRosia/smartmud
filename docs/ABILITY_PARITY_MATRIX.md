# Ability Parity Matrix

> **Audit state — evidence-bounded, not a parity claim.** The requested customized Adventurer's Lair checkout is not present in this container and direct GitHub access fails with `CONNECT tunnel failed, response 403`. Consequently, this is the authoritative **Smart MUD-side migration inventory and source-evidence ledger**, not an assertion that hidden customized legacy abilities have been exhaustively observed. A missing legacy fact is written as `Not legacy-verified`/`UNKNOWN`, never guessed. Importing the legacy source and rerunning the source-discovery checklist is the blocking acceptance criterion.

## Scope and counting rules

* **73 current authored definitions:** 59 canonical world definitions in `worlds/shattered_realms/abilities/abilities.json` and 14 core prototype definitions in `data/core_game/abilities/spells.json`. Prototype definitions are tracked separately because they are content seeds, not automatically live grants.
* An **ability** is a player/NPC/script/action capability with gameplay consequences. Commands which activate capabilities, passives, runtime-created builder definitions, scripts, quests, and monster specials each receive a row or a bounded dynamic family row.
* `IMPLEMENTED` requires an executable canonical path and the Phase 19/20 runtime; `PARTIALLY_IMPLEMENTED` means metadata/foundation exists but full legacy behavior is not proven; `NOT_STARTED` means no executable ability behavior; `BLOCKED_BY_DEPENDENCY` is reserved for dynamic families awaiting a listed runtime; `UNKNOWN` means source evidence is unavailable. Every row has exactly one status.

## Per-row field contract

The compact table is the planning view. The canonical JSON definition named in **Current implementation** is the field-level record for primary function, classes, unlock, resource/HP/mana/move cost, cooldown/wait, target/position/equipment/weapon restrictions, formulas, duration/stacking, affects, messages, AI hints, and builder metadata. Values absent from that definition are **not legacy-verified**; they are not inferred from a name. No ability has an established HP cost unless its definition says so. Death/corpse, room/zone/world, persistence, summon/object, and script interactions are likewise explicitly `Not legacy-verified` unless shown by the dependency/status notes.
