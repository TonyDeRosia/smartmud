# Phase 14A Adventurer's Lair Ability/Effect Parity Audit

Network restrictions prevented a local clone (`CONNECT tunnel failed, response 403`), so the audit used GitHub web/raw access to the requested Adventurer's Lair files and the checked-out Smart MUD repository as authoritative implementation context. This document records behavior, not copied code, tables, formulas, or file formats.

| Behavior | Adventurer's Lair reference file/function | Player experience | NPC/mob behavior | Current Smart MUD equivalent | New canonical Smart MUD path | Status | Intentional difference |
|---|---|---|---|---|---|---|---|
| Spell/skill registration | `src/spell_parser.c` `spell_info`, `spello`, `skillo`; `src/class.c` class practice maps | Spells/skills have names, minimum levels, mana, targets, routines, and wear-off metadata. | NPCs invoke spells through the same registered numbers but can be scripted/mobile driven. | `engine.abilities.AbilityRegistry` loads world `abilities`. | `AbilityDefinition` now has Phase 14A fields plus canonical effect operations in `plugin_data.canonical_effects`. | Implemented foundation. | IDs are string/world-authored rather than numeric slots. |
| Lists | `do_skills`, `do_spells`, `do_practice` | Players see learned skills/spells and practice percentages. | NPCs do not need player learning rows. | `abilities`, `skills`, `spells` commands use ability service/display code. | `AbilityAvailabilityService.resolve_actor_abilities` merges learned/player rows and NPC template grants. | Implemented foundation. | Display remains Smart MUD semantic display, not C text tables. |
| Learned proficiency/practice | `do_practice`, class trainers, skill percentages | Practice spends sessions and raises percent up to caps. | NPC templates effectively define competence directly. | `actor_ability_progression`, `attempt_proficiency_improvement`. | `ActorAbilityProficiency` immutable model and availability output proficiency. | Partial. | Trainer conversations remain Phase 9A; Phase 14A adds model/foundation. |
| Improvement through use | skill/spell success hooks and improve references | Repeated valid use can improve learned percent. | Mobs do not grind practice. | Deterministic improvement hook in `AbilityExecutionService`. | Same pipeline, no invalid-use improvement. | Implemented foundation. | Deterministic test hook instead of C random formula. |
| Targets | `do_cast`, `call_magic`, target flags | Self, actor, object, room/group targets determine legality. | Mob AI passes targets to same spell routines. | `resolve_target` modes. | `TargetingDefinition` and one resolver for cast/use/invoke/perform/NPC callers. | Implemented foundation. | Phase 14B can add richer geometry. |
| Resource costs | mana cost in spell parser/cast flow | Mana/stamina consumed only for legal actions. | NPCs spend resources unless authored otherwise. | `RuntimeResourceService.pay_cost`. | `AbilityCostDefinition`; execution validates resources before payment. | Implemented foundation. | Uses canonical resource service and idempotent action direction. |
| Materials/reagents | `mag_materials` | Reagents can be required and consumed. | NPC/script behavior can require components. | Previously no canonical ability material transaction. | Ability `plugin_data.materials` validates and consumes canonical `item_instances` atomically. | Implemented representative. | Item alternatives/groups reserved for expansion. |
| Casting restrictions/posture | `do_cast`, position checks | Dead/sleeping/stunned/etc. block casting. | Mobs obey unless special routines bypass. | `validate_ability_use` has posture/combat checks. | Prereq model documents canonical posture/lifecycle restrictions. | Partial. | Some states remain existing lifecycle integration work. |
| Direct damage | `mag_damage`, fight damage path | Damage spell rolls save/resistance/critical and reports damage. | NPC spells damage players/mobs through same combat. | Damage components call combat runtime when available. | `deal_damage` operation builds canonical combat action request. | Implemented foundation. | Existing `CombatRuntimeService` finalizes damage. |
| Healing | `mag_points` | Healing restores HP but normal heal does not resurrect. | NPC healers can heal. | `apply_healing` uses `RuntimeResourceService`. | `heal`/HOT use canonical resource/healing service. | Implemented foundation. | Critical heal scaling hook remains via combat/stat services. |
| Saves | saving throw calls in magic/fight | Victims can negate/reduce hostile effects. | NPCs roll same save categories. | Combat runtime has save definitions for damage. | Effect definitions carry `physical_save`, `mental_save`, `magic_save`-style metadata. | Partial. | Save metadata is preserved; deeper alternate outcomes are Phase 14B. |
| Buffs/debuffs/duration | `mag_affects`, `affect_to_char`, `affect_join` | Timed modifiers alter stats until expiration/wear-off. | Same affect list on mobs. | `actor_effect_instances`, Phase 5E modifiers. | `RuntimeEffectInstance`, canonical operations persist active effects. | Implemented foundation. | Modifiers are normalized JSON for stat authority. |
| Wear-off | `wear_off_msg`, `affect_remove` | Message appears when effect expires. | Mob effects clean up too. | Effect table supports metadata. | Expiration processes once and returns authored wear-off messages. | Implemented foundation. | Presentation fan-out remains command/runtime layer. |
| Stacking/replacement/exclusivity | `affect_join`, protective flags, mutually exclusive affects | Recasts refresh/stack/replace depending spell. | Same. | Existing effect templates had policies. | Stack/exclusive metadata validated and persisted. | Partial. | Generic groups replace hardcoded spell names. |
| Poison/disease/blind/sleep/charm/haste/slow/sanctuary | `mag_affects`, constants, affect flags | Control/condition flags restrict actions or alter stats. | Mobs can be affected if not immune. | Effect templates and lifecycle states. | `apply_condition`, DOT, cleanse/dispel, grant resistance/immunity operations. | Representative implemented. | Full AL catalog intentionally out of scope. |
| Dispels/cures | `mag_unaffects`, cure/dispel spells | Removes matching harmful/beneficial effects, with protection rules. | Same. | Limited effect removal. | `dispel` and `cleanse` remove by tag/category from persistent effects. | Implemented foundation. | Difficulty/resisted dispel metadata reserved. |
| Group/area effects | `mag_groups`, `mag_masses`, room routines | Group/room targets receive effects. | NPCs can use mass spells. | `room_*` target modes exist. | Target kinds include group/area/room. | Foundation. | Representative tests focus single targets. |
| Cleanup on death/logout/restart | handler/limits affect cleanup, persistence policies | Temporary affects drop on death/logout depending policy; persistent state recovers after restart. | Same. | `actor_effect_instances` persists. | `process_effect_expirations`, tick claims, death/logout policy fields in metadata. | Partial. | Conservative no-duplicate tick recovery implemented; full policy matrix future. |
