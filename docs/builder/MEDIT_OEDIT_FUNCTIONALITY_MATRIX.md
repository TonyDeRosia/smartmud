# MEDIT/OEDIT Functionality Matrix — Phase 15B.38.1

## Reference audit

Local search for `tbamud_adventurers_lair`, `tbamud`, and `adventurers_lair` found no customized TBA source tree in `/workspace`; available checked-in parity docs inspected were `docs/OBJECT_EDITOR.md`, `docs/ROOM_EDITOR.md`, `docs/ZONE_EDITOR.md`, `docs/TBAMUD_COMMAND_PARITY.md`, and `docs/SMART_MUD_TBAMUD_COMMAND_AND_DISPLAY_AUDIT.md`. The unverified-reference item is exact customized C source behavior for Oasis MEDIT/OEDIT submenus; this phase therefore reproduces the documented TBA-style workflow (numbered menus, temporary draft structures, explicit save, validation, lookup, preview) through Smart MUD canonical drafts and services.

## Functional subeditor standard

Structured editors must enter a submenu, render current entries with readable labels and canonical IDs, support add/edit/remove/clear where applicable, support search/select for referenced records, preview, validate, undo, redo, cancel/back, dirty-state tracking, and save only through the parent Builder draft. User-facing MEDIT/OEDIT workflows must not require raw JSON, Python dictionary syntax, bitvectors, or serialized reset/loadout strings.

## Options converted from shell to structured subeditors

* MEDIT Equipment loadout: converted from a single `equipment_loadout` raw prompt to a structured equipment/carried manager.
* MEDIT Spawn/reset references: converted from `set/add/remove <field> <value>` command shell to a structured spawn-reference manager.
* OEDIT menu visibility: type-specific object sections are conditionally displayed instead of implying all subsystems always apply.

## Hidden or unavailable options

* OEDIT container/light/food/drink sections are hidden unless the object type supports them.
* Script/trigger attachment remains limited where no canonical script registry data exists in the selected world package; no fake script language was added.
