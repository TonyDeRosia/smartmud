# Phase 13C1F Display Parity Audit

This audit records the Smart MUD implementation against the Adventurer's Lair player-facing display model without copying source, structs, tables, or file formats.

| Category | Adventurer's Lair behavior | Smart MUD canonical source | Smart MUD visual implementation | Intentional difference | Tested | Builder customizable |
|---|---|---|---|---|---|---|
| SCORE | Dense fixed-width score with identity, resources, attributes, combat, carry, currency, effects, and time. | `CharacterDisplaySnapshotService` gathers progression, actor resources, calculated stats, inventory/carrying, currency, survival, visible effects, and time. | `build_score_document()` formats an immutable snapshot into framed semantic rows. | Unsupported gameplay values are omitted instead of fabricated as zero. | Focused character display tests and compile checks. | Width, labels, section lists, roles, borders via display theme contracts/drafts. |
| WORTH | Shows player-facing coin/currency totals. | Same currency snapshot used by SCORE. | `build_worth_document()` consumes `CharacterDisplaySnapshot.currency`. | Future/premium/ledger identifiers are not exposed. | Focused display tests. | Labels and role overrides. |
| Attributes | Shows base-versus-modified stats. | Actor attributes / ability score mappings normalized by snapshot service. | Attribute rows show base/modifier/final when available. | Missing classic-six stats are omitted. | Focused display tests. | Section visibility/order and labels. |
| Resists/combat | Shows canonical combat defenses/offense. | Calculated stats mapping, formula-facing actor stats where available. | Combat rows in SCORE; focused commands remain routed through score sections. | Unimplemented values are omitted. | Compile and focused command tests. | Section visibility/order and labels. |
| Skills/spells/abilities | Lists learned abilities with status, cost, rank, target/cooldown context. | `AbilityDisplaySnapshotService` plus `AbilityExecutionService.validate_ability_use()`. | `build_abilities_document()` renders SKILLS/SPELLS/ABILITIES frames. | Targeted abilities can report ready-needs-target rather than blocked. | Compile and focused display tests. | Width, role, labels, prompt/default theme data. |
| Cooldowns | Shows active cooldowns in natural time. | Ability cooldown table via `AbilityExecutionService` validation/cooldown status. | COOLDOWNS frame lists active cooldown rows. | Raw timestamps are not shown. | Compile checks. | Labels/roles/width. |
| Affects | Shows visible effects with duration/stacks. | Snapshot service and affects command filter hidden/secret/admin-only records. | AFFECTS sections list visible classification, duration, stacks. | Secret/admin effects are omitted. | Compile checks. | Display family supported in theme validation. |
| Inventory/equipment | Shows carried/equipped player items with grouping and slots. | Runtime inventory/equipment lists and carrying snapshot for prompt/score agreement. | Existing inventory/equipment documents remain semantic. | No fake carry values if absent. | Existing command tests. | Theme validation supports families. |
| Prompt | Configurable token prompt with presets and custom safe templates. | Character preferences and snapshot-compatible token fields. | `build_prompt_document()` renders semantic prompt segments. | Malformed tokens render warning or are rejected by command validation. | Prompt command tests and display tests. | Theme prompt presets and player preferences. |
| Prompt tokens | Classic HP/MP/ST, XP/TNL, gold, level, alignment, posture, target, room, zone, quest, time, name, age, play time, encumbrance, weapon, combat. | CharacterDisplaySnapshot-compatible character/runtime fields. | Token scanner replaces known tokens and supports `%%`. | Hidden IDs are not intentionally exposed. | Focused prompt tests. | Prompt templates/presets. |
| Display themes | Builder-authored visual customization. | World package `display_themes` plus validated runtime resolver. | `ResolvedDisplayTheme` normalizes width, labels, roles, borders, section data, templates, prompt presets. | Current live builder support is draft/in-memory unless persisted by existing builder export flow. | Compile checks and command smoke coverage. | Yes. |
